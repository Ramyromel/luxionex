Reporting a Vulnerability

If you discover a security vulnerability, please do not create a public GitHub issue.

Instead, report it privately by using one of the following methods:

- GitHub Security Advisories (preferred)
- Private security contact (if available)

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- A proof of concept (if applicable)
- Suggested mitigation (optional)

---

Response Process

After receiving a report we will:

1. Confirm receipt within 72 hours.
2. Reproduce and validate the issue.
3. Assess severity and impact.
4. Develop and test a fix.
5. Publish a security advisory after the fix is available.

---

Scope

This policy applies to:

- Core platform
- AI execution engine
- Connectors
- Agent framework
- Security layer
- Registry system
- Routing engine
- Decision engine

Third-party dependencies are handled according to their respective maintainers.

---

Responsible Disclosure

Please allow sufficient time to investigate and resolve reported vulnerabilities before making them public.

We appreciate responsible disclosure and collaboration from the security community.
